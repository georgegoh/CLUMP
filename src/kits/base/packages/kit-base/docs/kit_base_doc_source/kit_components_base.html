Included components

This is a mandatory OCS kit.

The Platform OCS Base kit is made up of two main components:

  • component-base-installer: This component provides the primary install node with various Kusu management tools. By default, it is associated with the install nodegroup.

  • component-base-node: This component provides nodes in the OCS cluster with the Kusu runtime for non-install-type nodes. By default, it is associated with compute, compute-imaged, and compute-diskless nodegroups.

Enable components

Using the Node Group Editor (ngedit), components can be individually enabled or disabled within a nodegroup.

  1. Run ngedit.
  2. After selecting the nodegroup on which this kit is installed, navigate to the Components window and select base.
  3. Select (or clear) the desired kit components.

    By default, component-base-installer is enabled on the OCS primary install node, while component-base-node is enabled on all compute nodes in the cluster.
