About this kit

This Base kit contains a number of useful tools and utilities to help make cluster management easier.

Tool

Description

addhost

This tool lets you quickly and easily add and remove nodes from a specified nodegroup. It also performs the following functions:

  • Monitors the DHCP log file for new MAC addresses; when a new address is detected, the host is added to the specified nodegroup

  • Updates existing hosts with new MAC addresses—this allows you to replace a compute node if it fails

  • Adds multiple hosts using a a file that contains a list of MAC addresses

  • Adds unmanaged nodes to the cluster

boothost

This tool creates the PXE configuration files used when starting up a node. Using this tool you can perform the following:

  • View the boot status of nodes or nodegroups

  • Set kernel and initrd parameters for a host

  • Write PXE files to /tftpboot/kusu/pxelinux.cfg (each node has a PXE file that is created using the node’s MAC address)

  • Reinstall nodes using pdsh

buildimage

This tool creates images (diskless and imaged). Other attributes include the following:

  • Automatically called by ngedit

  • Used in scripts to create custom Kusu tools

  • Run when packages are added or removed from an image

  • Creates a chrooted file system to assemble all of the packages for an image

buildinitrd

This tool builds the initial ram disk for diskless and imaged nodegroups. Other attributes include the following:

  • Automatically called by ngedit

  • Used to customize the ram disk to support hardware or new drivers for existing hardware

  • Run this to rebuild the initrd when kernel drivers are added or removed from a nodegroup

  • Stores initrds in /tftpboot/kusu

cfm

This tool is a configuration file manager that synchronizes nodegroup packages and files. It also has the following attributes:

  • Automatically called by ngedit and addhost tools

  • Extensible and can manage any file, including binaries

  • Restarts services on file changes by using plug-ins

  • Wraps other tools, such as cfmsync (prepares files for distribution by the web server and broadcasts when available), cfmd (listens for the cfmsync broadcast and then starts cfmclient), and cfmclient (downloads appropriate packages from the http server)

netedit

This tool defines available network interfaces in the cluster; these defined networks are then associated with nodegroups. Netedit can define any network, including those not connected to the primary install node. Defined networks include Ethernet and Infiniband.

ngedit

This is the nodegroup editor tool for Kusu. (A nodegroup is a template that defines cluster node properties, including repositories, install-type, components, network configurations, OS packages, custom scripts, etc.)

nghosts

This tool moves hosts between node groups. Choose to move individual hosts of entire nodegroups.

repoman
Nodegroups in an Kusu cluster are associated with a particular repository—This tool manages those repositories. Note the following:

  • Kusu kits are added to repositories

  • Multiple repositories can be installed on a single Kusu primary install node

  • Repositories can be customized and updated; repository snapshots can be taken

  • OS media kits are turned into repositories.

repopatch

This tool updates Kusu repositories using yum, and/or from RHN.

kitops

This tool is used to manipulate kits in an Kusu cluster. It can add, remove, and list kits. (Kits are pre-packaged applications or services that can be automatically installed and configured on cluster nodes once installed in an Kusu cluster).

Note the following:

  • Kits can be installed from an iso, or a mount point

  • Kits must be added to a repository after they are installed

  • Kits typically contain default nodegroup associations

boot-media-tool

This tool is used by developers and Platform support personnel to create an Kusu Kusu boot media from supported linux distributions.

buildkit

This tool allows administrators and developers to build new Kusu kits. It does this by creating a template workspace for the kit.

driverpatch

This tool updates the initrd image with new sets of driverpacks.

genconfig
This is the Kusu database reporting and file generating tool. It creates these configuration files for system services:

  • debug: A debug file for support purposes

  • hosts: The cluster’s /etc/hosts file

  • hostspdsh: The cluster hosts file for pdsh

  • zone: The DNS zone file for the cluster

  • named: The DNS named.conf file

  • nodegroups: A list of Kusu nodegroups

  • bashrc: The cluster bashrc configuration file

  • nodes: A list of nodes in the cluster

  • apache_conf: An apache configuration file

  • hostequiv: The hostequiv file

  • dhcpd: The dhcpd configuration file

  • revers: The DNS reverse lookup file

  • ssh: The ssh configuration file for nodes

Some plugins for this tool may allow the generation of configuration files for other Kusu kits such as LSF, Lava, Nagios, and Cacti.

This tool is automatically called other Kusu tools such as addhost and ngedit.

pdsh

This high-performance, parellel remote shell utility is used to run commands in parallel across the Kusu cluster. Note the following:

  • Kusu automatically generates a pdsh “hosts” file

  • Configured to use ssh

  • Uses different remote execution modules, such as rsh, ssh, exec, etc.

This kit also provides an Environment Module package. Use this package to dynamically modify your environment with the use of modulefiles. Every modulefile contains the information required for configuring an application shell.
